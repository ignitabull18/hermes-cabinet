# אפליקציות ו-Repos

Cabinet הולך מעבר לדפי markdown. אפשר להטמיע אפליקציות web שלמות, לקשר ל-repositories של Git, וליצור כלים אינטראקטיביים שחיים לצד התיעוד.

## אפליקציות מוטבעות

כל ספרייה שמכילה קובץ `index.html` **ואין בה** `index.md` נחשבת כאפליקציה מוטבעת. Cabinet מציג אותה ב-iframe.

### אפליקציה מוטבעת רגילה

האפליקציה מרונדרת באזור התוכן הראשי, סרגל הצד וחלונית ה-AI עדיין גלויים. מתאים ללוחות מחוונים וכלי עזר שמסתכלים עליהם תוך כדי עבודה.

```
data/
  my-dashboard/
    index.html     ← האפליקציה (מרונדרת כ-iframe עם סרגל הצד)
    app.js
    style.css

```

### אפליקציה במסך מלא (סימן `.app`)

הוספת קובץ סימון ריק בשם `.app` לספרייה — האפליקציה מקבלת מקסימום מקום: סרגל הצד וחלונית ה-AI מתכווצים אוטומטית בפתיחה. כפתור **חזרה ל-KB** בסרגל הכלים משחזר את הפריסה הרגילה.

```
data/
  my-tool/
    index.html     ← האפליקציה
    .app           ← סימן: מצב מסך מלא
    other-files/

```

שני הסוגים מופיעים בסרגל הצד אוטומטית — בלי build, בלי deployment.

## Repositories מקושרים

קובץ `.repo.yaml` בכל ספריית data מקשר אותה ל-Git repository. סוכנים משתמשים בקובץ הזה כדי לקרוא ולחפש בקוד מקור כשהם עובדים על תיעוד קשור.

```yaml
name: my-project
local: /path/to/local/repo
remote: https://github.com/org/repo.git
source: both
branch: main
description: מה ה-repo הזה מכיל (עוזר לסוכן להבין הקשר)

```

**שדות:**

| שדה | חובה | תיאור |
| --- | --- | --- |
| `name` | כן | שם פרויקט קריא |
| `local` | כן | נתיב מוחלט ל-clone המקומי |
| `remote` | לא | URL של GitHub — לקישורים, issues, הצעות PR |
| `source` | לא | `local`, `remote`, או `both` (ברירת מחדל: `both`) |
| `branch` | לא | branch ברירת מחדל (ברירת מחדל: `main`) |
| `description` | לא | תיאור חופשי להקשר של הסוכן |

כשסוכן עובד על דף KB שיש בו `.repo.yaml` באותה ספרייה או באחת מההורים, הוא:

1.  קורא את `.repo.yaml` כדי למצוא את ה-repo המקושר
2.  משתמש בנתיב ה-`local` לקריאת קוד והבנת ארכיטקטורה
3.  משתמש ב-URL ה-`remote` ליצירת קישורים או הצעת PRs

סרגל הצד מציג את הספריות האלה עם אייקון **GitBranch** כתום.
