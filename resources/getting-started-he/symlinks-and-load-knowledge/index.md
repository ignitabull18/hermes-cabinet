# סימלינקים וטעינת ידע

Cabinet משתמש בסימלינקים ישירים כדי להביא תיקיות חיצוניות אל מאגר הידע שלכם בלי להעתיק שום דבר. התיקייה נשארת היכן שהיא בדיסק — Cabinet רק יוצר מצביע אליה.

## איך טעינת ידע עובדת

1. לחיצה ימנית על פריט כלשהו בסרגל הצד
2. בחירה ב-**טעינת ידע**
3. בחירת תיקייה במחשב (או הדבקת הנתיב)
4. אופציונלי: קביעת שם תצוגה
5. לחיצה על **טעון**

Cabinet יוצר סימלינק בתוך ה-KB:

```
data/my-project -> /Users/me/Projects/my-project
```

תוכן התיקייה מופיע ישירות כילדים בעץ סרגל הצד. בלי תיקיות עטיפה, בלי הקשת נוספת.

## מה נכתב

Cabinet כותב שני קובצי dot מוסתרים אל **תיקיית היעד**:

### `.cabinet-meta`

כל תיקייה מקושרת מקבלת את זה. שומר מטא-דאטה לתצוגה ב-KB. מוסתר מסרגל הצד.

```yaml
title: My Project
tags:
  - knowledge
created: '2026-04-12T00:00:00.000Z'
```

> Cabinet גם קורא קבצי `.cabinet.yaml` ישנים כשהם קיימים, אבל קישורים חדשים משתמשים ב-`.cabinet-meta`.

### `.repo.yaml` (רק עבור git repos)

אם התיקייה היא git repo, Cabinet מזהה אוטומטית את ה-branch וה-remote וכותב `.repo.yaml` כך שסוכני AI יוכלו לקרוא את קוד המקור בהקשר.

```yaml
name: my-project
local: /Users/me/Projects/my-project
remote: https://github.com/me/my-project.git
source: both
branch: main
```

אם `.repo.yaml` כבר קיים בתיקייה, Cabinet מדלג על הכתיבה.

שני קבצי ה-dot מוסתרים מסרגל הצד כברירת מחדל.

## אייקונים בסרגל הצד

| אייקון | צבע | משמעות |
|------|-------|---------|
| GitBranch | כתום | git repo מקושר (יש `.repo.yaml`) |
| Link2 | כחול | ספרייה מקושרת שאינה repo |

## ביטול קישור

להסרת תיקייה מקושרת:

1. לחיצה ימנית על הפריט המקושר בסרגל הצד
2. בחירה ב-**בטל קישור**

זה מסיר רק את הסימלינק מה-KB. התיקייה המקורית וכל הקבצים שלה לא נוגעים בהם. ה-`.cabinet-meta` בתיקיית היעד גם מנוקה.

## שינוי ספריית הנתונים

כברירת מחדל, Cabinet שומר תוכן ב-`./data` (מצב dev) או בנתיב app-data ספציפי לפלטפורמה (Electron). אפשר לדרוס עם משתנה הסביבה `CABINET_DATA_DIR`:

```bash
CABINET_DATA_DIR=/path/to/my/kb npm run dev
```

אפשר גם לעשות את `data/` עצמה סימלינק שמצביע למקום אחר — בונה העץ עוקב אחרי סימלינקים בשקיפות.

## טיפים

- תיקיות מקושרות עם `index.html` (ובלי `index.md`) מרונדרות כאתרים מוטבעים
- הוסיפו סימן `.app` למצב מסך מלא
- אם לתיקיית היעד יש `index.md` משלה, Cabinet משתמש בה כדף הנחיתה
- סוכנים מגלים repos מקושרים על-ידי קריאת `.repo.yaml` בספרייה הנוכחית או בכל אחת מההורים
