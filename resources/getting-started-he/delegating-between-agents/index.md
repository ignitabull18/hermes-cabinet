# שיגור בין סוכנים

סוכן ב-Cabinet אינו רק מפעיל עצמאי — הוא יכול להציע עבודה לחבריו לצוות. כשמנכ״ל מתכנן השקה, הוא יכול לשגר מחקר לאנליסט, ניסוח לעורך, ולתזמן עדכון שבועי לצמיחה — הכול בתור אחד. כל הצעה ממתינה לאישור אנושי בלחיצה אחת לפני שמשהו מתחיל, כך שאתם בשליטה על עץ העבודה.

## הפעלת שיגור

פתחו סוכן (לחיצה בסרגל הצד או מעבר ל-`/agents/<slug>`). ליד החלפת **פעיל** בכותרת, הפעילו את **יכול לשגר**.

- **יכול לשגר** (מלא) — הסוכן רשאי להציע פעולות `LAUNCH_TASK`, `SCHEDULE_JOB`, ו-`SCHEDULE_TASK`.
- **אין שיגור** (מקווקו) — כל שיגור שהסוכן פולט מסומן באזהרה `persona_cannot_dispatch` ונחסם.

הובלות מופעלות כברירת מחדל; שאר הסוכנים זה opt-in. הפעילו עבור סוכנים ספציפיים שתרצו שיהיו מתזמרים (טיפוסי: מנכ״ל, PM, ראש פרויקט).

## איך סוכן מציע עבודה

מאחורי הקלעים, הסוכנים עוטפים הצעות בבלוק `cabinet` בסוף התשובה. Cabinet מנתח את אלה ללא צורך בכלים מיוחדים. שתי צורות נתמכות:

**Inline — פעולה לשורה:**

```cabinet
LAUNCH_TASK: <agent-slug> | <title> | <one-line prompt>
SCHEDULE_TASK: <agent-slug> | <ISO datetime> | <title> | <prompt>
SCHEDULE_JOB: <agent-slug> | <name> | <cron> | <prompt>
```

**JSON — פרומפטים רב-שורתיים או פיזור גדול (מעל ~5 פעולות):**

````markdown
```cabinet-actions
[
  { "type": "LAUNCH_TASK", "agent": "<agent-slug>", "title": "<title>", "prompt": "<prompt>", "effort": "high" },
  { "type": "LAUNCH_TASK", "agent": "<agent-slug>", "title": "<title>", "prompt": "<prompt>", "effort": "low" }
]
```
````

Cabinet מזהה כפילויות לפי טביעת אצבע (type + agent + title + prompt + runtime), כך שאם סוכן חוזר על עצמו בטעות, אותה שורה לא תוצג פעמיים.

## שלושת סוגי הפעולות

| פעולה | מתי רצה | להשתמש ל |
| --- | --- | --- |
| `LAUNCH_TASK` | מיד עם אישור | האצלה חד-פעמית — ״עורך, נסח את זה עכשיו״ |
| `SCHEDULE_TASK` | פעם אחת, ב-ISO datetime ספציפי | ירייה ושכח מתוזמן — ״רוץ ביום שני ב-9״ |
| `SCHEDULE_JOB` | חוזר, על cron | פעימה קבועה — ״כל יום שני 09:00, סכם את השבוע שעבר״ |

`SCHEDULE_TASK` שיורה תוך פחות מ-60 שניות מקוצר ל-`LAUNCH_TASK` רגיל — אין טעם לנתב ״בעוד 30 שניות״ דרך cron.

## אישור הצעות

כשהסוכן המשגר מסיים את התור שלו, **פאנל פעולות ממתינות** מופיע מתחת לתגובה הסופית. לכל שורה תקבלו:

- צ׳ק-בוקס לכלול או להחריג את השורה מהאצווה.
- סוג הפעולה (ורוד ל-launch, כחול ל-schedule-job, סגול ל-schedule-task).
- הסוכן היעד, הכותרת, ותצוגה מקדימה של ה-prompt.
- **אזהרות**: `unknown_agent`, `persona_cannot_dispatch`, `self_dispatch`, `cycle_risk`, `depth_warning`, `inactive_target`, `budget_low`, `invalid_schedule`, `invalid_when`. אזהרות קשות חוסמות את השורה; אזהרות רכות מאפשרות להמשיך אם אתם מקבלים את הסיכון.
- תפריטי **מודל** ו-**מאמץ** מוטבעים (פירוט בהמשך).

השתמשו ב-**אשר נבחרים**, **אשר הכול** (מדלג על שורות חסומות-קשה), או **דחה הכול** כדי לרוקן את התור. אישור יוצר שיחות-בנים מתויגות עם `parentTaskId`, `triggeringAgent`, ו-`spawnDepth` ב-`meta.json` שלהן, כך שאפשר לעקוב אחורה אל המקור.

## הצמדת מודל ומאמץ לתת-משימה

כברירת מחדל, שיחה-בת בוחרת מודל ומאמץ לפי סדר העדיפות:

1.  **דריסה של הסוכן המשגר** — אם הוא הציב `model` / `effort` בפעולה עצמה.
2.  **ירושה מהאב** — אם ספק שיחת ההורה תואם לזה של היעד, ה-`model` וה-`effort` של ההורה מועתקים אל הילד. מנכ״ל Claude-Opus שמשגר לסוכן Claude אחר מקבל ילד Opus בלי שמישהו ביקש.
3.  **ברירות מחדל של פרסונת היעד** — ה-`adapterConfig` של חבר הצוות ב-`persona.md`.

מעבר בין ספקים (הורה Claude → ילד Codex) משמיט את המודל/מאמץ של ההורה אוטומטית — בלי דליפה בין ספקים. ספק היעד והאדפטר שלו **תמיד** מנצחים; רק רמת החשיבה עוברת.

### דריסה בעת אישור

לכל שורה בפאנל הפעולות הממתינות יש שני תפריטים קטנים:

- **model** — בחירה מהקטלוג של ספק ההורה, או ״השתמש בברירת מחדל״ כדי לתת לירושה לרוץ.
- **effort** — בחירה ב-`minimal` / `low` / `medium` / `high` / `xhigh` / `max`, או ״השתמש בברירת מחדל״.

בחרו מה שאתם רוצים לפני האישור. הדריסה נכתבת אל ה-`adapterConfig` של הילד שנוצר.

### דריסה בפעולה עצמה

סוכנים יכולים להצמיד runtime ישר בתוך ההצעה:

**Inline:** צרפו `| model=<m>` ו/או `| effort=<e>` כסגמנטים נגררים. סגמנטים `key=value` לא מוכרים נחשבים כטקסט prompt, אז בטוח להשמיט את שניהם.

```cabinet
LAUNCH_TASK: <agent-slug> | <title> | <one-line prompt> | effort=high
LAUNCH_TASK: <agent-slug> | <title> | <one-line prompt> | model=<model> | effort=low
```

**JSON:** הוסיפו מפתחות `model` ו/או `effort`.

```json
{ "type": "LAUNCH_TASK", "agent": "<agent-slug>", "title": "<title>", "prompt": "<prompt>", "effort": "high" }
```

ה-prompt של המנכ״ל המשגר כבר מזכיר לכל סוכן מאופשר-שיגור על השדות האלה, אז אפשר פשוט לומר ״השתמש ב-effort גבוה לעורך״ בבקשה והסוכן יקודד את זה.

## קשור

- מסגרת סוכנים: ראו את **לוח בקרת סוכנים** ו-**משימות מתוזמנות** ב-[[Getting Started]].
- הגדרות פרסונה: `.agents/<slug>/persona.md` — קבעו שם `canDispatch`, `provider` ברירת מחדל, `adapterType`, ו-`adapterConfig`. הפעלת **יכול לשגר** בכותרת כותבת חזרה לקובץ הזה.
