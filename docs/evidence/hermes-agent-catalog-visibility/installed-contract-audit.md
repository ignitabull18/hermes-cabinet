# Hermes Agent catalog contract audit

Audit date: 2026-07-20. Installed live runtime: Hermes Agent `0.19.0`. Installed Agent checkout: `d7b36070ef807841699ad32c5b6af547fee3ff64` (`v2026.7.20-28-gd7b36070e`). Installed source authority: `~/.hermes/hermes-agent/gateway/platforms/api_server.py`.

The authenticated loopback Agent server uses `aiohttp`. Its `/openapi.json` route is unavailable, so the exact contract authorities are the installed source and authenticated `GET /v1/capabilities` response. No Management or Gateway credential was configured. No service was restarted and no Hermes mutation was performed.

| Interface | Method and installed response | Authentication, timeout, and error boundary | Projected safe fields | Removed fields and limitations |
| --- | --- | --- | --- | --- |
| `/v1/capabilities` | `GET`; capability object listing stable Agent routes, including Skills and Toolsets | Server-only bearer API key; bounded timeout; redirects refused; source-specific outcome | Contract identity and observation status only | Nested capability metadata is not returned to the browser. |
| `/v1/skills` | `GET`; `{ object: "list", data: Array<{ name, description, category }> }`; no pagination contract | Server-only bearer API key; bounded timeout; redirects refused; authentication, unavailable, failure, empty, and success remain distinct | Bounded skill display identity, name, category, total count, duplicate count, truncation state, source, interface, and observation time | Description and arbitrary fields are discarded. The endpoint does not explicitly report per-skill enabled state or provenance, so visibility is a partial claim and cannot earn canonical Skills live parity. |
| `/v1/toolsets` | `GET`; `{ object: "list", platform: "api_server", data: Array<{ name, label, description, enabled, configured, tools }> }`; no pagination contract | Server-only bearer API key; bounded timeout; redirects refused; authentication, unavailable, failure, empty, and success remain distinct | Bounded display identity, label, explicit enabled/configured state, tool count, bounded platform provenance, total count, duplicate count, truncation state, source, interface, and observation time | Description, concrete tool names, schemas, commands, URLs, paths, credentials, and arbitrary metadata are discarded. Catalog presence does not prove Executor operational health or canonical API-key configuration. |

Collector bounds are 200 Skills and 100 Toolsets. Duplicate identities are selected deterministically by completeness and then by the safe projected representation; discarded raw metadata is not used as a browser-visible tie-breaker. Unsafe labels, URLs, paths, authorization text, shell-like content, control characters, and oversized strings are rejected or normalized before projection.

All reads use `cache: no-store`, `redirect: error`, a source-specific abort timeout, and server-only authentication. The recursive projection sanitizer remains the final browser-model boundary. No description, concrete tool name, secret-bearing URL, local absolute path, credential value, raw response, or arbitrary metadata blob enters evidence or UI output.
